1 · Executive summary

Both graphs are memory-bound on DMA, and the root cause is the same: with batch=1 and decode routed through forward_all_experts, every forward pass streams all 256 experts of every MoE layer from HBM even though only the top-8 are used. In decode, expert weights are 90.4% of all HBM read traffic (16.5 GB/round); they alone impose a ~13.4 ms floor at peak LNC2 bandwidth, against the measured 29.95 ms round. Traffic is otherwise already minimal — no reloads or spills — so the recovery levers are bandwidth utilization and idle gaps, and the only lever that moves the floor is shrinking expert bytes (sparse expert loading or weight quantization).

Prefill is the same story at lower intensity: DMA active 64.7% of the 28.2 ms execution, with the MoE expert path at 62% of traffic, plus a packet-size problem decode doesn't have — 89% of prefill DMA packets are 1–4 KB, leaving a 12 ms gap between achieved and peak bandwidth. Compute (TensorE) is idle 50–70% in both graphs; DeltaNet and transposes only become relevant after the memory bound is fixed.

2 · How this was measured

A real generation run (16 new tokens, fused MTP speculation, greedy) on the pre-traced, pre-sharded model dumped per-graph NEFF+NTFF via NEURON_RT_INSPECT_ENABLE=1 / NEURON_RT_INSPECT_DEVICE_PROFILE=1. Three graphs appeared; the one-time layout_opt graph was discarded. Traces were ingested into neuron-explorer parquet; bounds use trn3 LNC2 constants (1228 GB/s aggregate peak HBM BW). The prefill trace holds ~11 steady-state executions (std < 0.1 ms); the decode trace holds exactly one fused round (41 MoE layers, each once).

5.6 · Inside the selective round — draft verification & waste analysis

The MTP draft head is in the graph and executes

Verified from the trace: the graph contains Qwen36MTPDraft (embed + RoPE, decoder layer [.42] = GroupQueryAttention_O[.12] + NeuronMoEBlock[.42], eh_proj concat, draft LM head ColumnParallelLinear[.257], argmax, KV write for layer 42) alongside Qwen36SpecTarget (40 verify layers). The draft runs first, in the ~559–1123 µs window — ~446 µs wall, 4.5% of the round, 48 MB HBM — which is why it's easy to miss next to the 40-layer verify (61.9% wall, 1.95 GB). In neuron-explorer, filter the layer column on Qwen36MTPDraft (the layer-name column carries module scopes; framework_name/ hlo_name are empty in these traces). The earlier "41 MoE layers" count = 40 target + 1 draft.

Total fully-idle time 1.49 ms (14.8%); gap to perfect pipeline 3.42 ms. Per-layer pipelining is healthy (40 layers contribute only 28 µs of inter-layer gaps) — the waste sits at phase boundaries and in collectives. Selective loading shifted the regime: the all-experts round was weight-streaming-bound; the selective round is serialization/overhead-bound.

Collectives detail

123 AllReduces (837 µs) + 7 AllGathers (133 µs). Each target layer fires exactly 3 reduces: attention/DeltaNet out_proj → routed-MoE → shared-expert down_proj (all 16 KB, ~7 µs each). The two MoE reduces are mathematically fusable — AR(routed) + gate·AR(shared) = AR(routed + gate·shared_local) in NeuronMoEBlock.forward — removing 41 ops (~190–250 µs exposed). The single worst collective is the target lm_head AllGather: 71 µs, 100% exposed, RDH algorithm — 12× the draft's identical-shape gather (7 µs, Mesh); switching to Mesh or keeping logits sharded into an on-device argmax saves ~57–71 µs.

Next optimizations for the selective build (ranked)

1 · Fill the draft→target bubble (~470 µs, ~4.4%). Prefetch the first verify layers' expert/attention weights during the draft pass — the DMA engines are completely idle for 534 µs after layer 42.
2 · Fuse the two MoE AllReduces per layer (~190–250 µs). Apply the sigmoid gate to the local shared-expert partial and add it to the routed partial before a single AllReduce in NeuronMoEBlock.forward.
3 · Overlap next-round weight prefetch with the vocab argmax (~550 µs DMA idle).
4 · lm_head gather: keep logits sharded into a sharded argmax (or force Mesh) (~60–70 µs exposed, plus the dominant share of transposes).
5 · De-serialize the accept/state-select tail (~160 µs) — the per-layer where/KV-scatter chain is gated by semaphore waits; batching state-select across layers would shorten the chain.

Combined ceiling ≈ 1.3–1.5 ms of the 10.0 ms round (~13–15%).

5.7 · Recommendations (ranked, original all-experts baseline)

1 · Stop streaming all 256 experts in decode — the only lever that moves the floor (est. ~2× decode). DONE — measured 2.9×
Decode was forced through forward_all_experts because selective loading hit an OOB DMA on an earlier code state (threshold zeroed in inference_qwen36_a3b.py). Re-enabled via A3B_SELECTIVE_DECODE=1 + recompile: the OOB does not reproduce on the fused graph; expert reads dropped ~32× per layer and rounds run 10.6 ms. See §5.5.
2 · Coalesce prefill's 1–4 KB dynamic DMAs (est. up to ~12 ms of prefill).
89% of prefill packets are 1–4 KB, holding effective BW far under peak — the gap between 18.2 ms achieved and 6.2 ms ideal. Larger contiguous tiles in the MoE dynamic-index/gather path (packets ≥16–64 KB) recovers several ms/execution.
3 · Close decode DMA idle + cross-core imbalance (est. 3–5 ms).
4.8 ms merged DMA idle (3.2 ms in 120 gaps >10 µs, clustered in the 29.1–30 ms tail drain); each core's DMA is idle ~37%. Deeper expert-weight prefetch and better cross-core co-scheduling keep both LNC2 cores' DMA saturated.
4 · Raise per-core DMA utilization (est. 1–3 ms decode). Active-window BW is ~728 GB/s of the 1228 GB/s aggregate peak; pushing expert blocks toward ≥64 KB transfers lifts utilization.
5 · Deferred (post-memory-fix): MoE layout transposes are 64% of TensorE FLOPs (5.6 ms) and DeltaNet's chunked-recurrence inner loop is prefill's largest compute consumer (~31 ms summed). Both cost ≈0 wall-clock today; revisit once DMA is no longer the bound and TensorE (14.7 ms decode / 8.3 ms prefill) becomes critical.
